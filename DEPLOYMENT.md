# StudyGen AI - Deployment Guide

Complete guide to deploy StudyGen AI to production on Vercel.

## Prerequisites

Before deploying, ensure you have accounts and keys for:
- **GitHub** - For repository hosting
- **Vercel** - For app deployment
- **Neon** - For PostgreSQL database
- **Stripe** - For payment processing
- **OpenAI** - For AI features
- **Vercel Blob** - For file storage

## Step 1: Prepare Local Repository

### 1.1 Initialize Git (if not already done)
```bash
cd your-project-directory
git init
git add .
git commit -m "Initial StudyGen AI setup"
```

### 1.2 Create GitHub Repository
1. Go to https://github.com/new
2. Create a new repository named `studygen-ai`
3. Push your local code:
```bash
git remote add origin https://github.com/yourusername/studygen-ai.git
git branch -M main
git push -u origin main
```

## Step 2: Set Up Integrations

### 2.1 Neon Database
1. Sign up at https://console.neon.tech
2. Create a new project
3. Note the connection string: `postgresql://user:password@host/dbname`
4. Keep this handy for Vercel setup

### 2.2 OpenAI API
1. Go to https://platform.openai.com/api-keys
2. Create a new API key
3. Copy and save securely

### 2.3 Stripe Setup
1. Sign up at https://dashboard.stripe.com
2. Create products for each plan:
   - **Pro Plan**: $19.99/month
   - **Enterprise Plan**: $99.99/month
3. Get price IDs from Stripe Dashboard
4. Generate webhook endpoint secret (we'll use this in Vercel)

### 2.4 Vercel Blob Storage
1. Go to https://vercel.com/dashboard/stores
2. Create a new Blob store named `studygen-ai`
3. Copy the `BLOB_READ_WRITE_TOKEN`

## Step 3: Deploy to Vercel

### 3.1 Import Project
1. Go to https://vercel.com/new
2. Click "Import Git Repository"
3. Paste your GitHub repository URL
4. Click "Import"

### 3.2 Configure Environment Variables
In the Vercel dashboard, add these environment variables:

**Database**
```
DATABASE_URL = [Your Neon connection string]
```

**Authentication**
```
BETTER_AUTH_SECRET = [Generated with: openssl rand -base64 32]
```

**AI Services**
```
OPENAI_API_KEY = [Your OpenAI API key]
```

**Stripe**
```
STRIPE_PUBLIC_KEY = [Your Stripe publishable key]
STRIPE_SECRET_KEY = [Your Stripe secret key]
STRIPE_WEBHOOK_SECRET = [Generated by Stripe - see below]
STRIPE_PRICE_ID_PRO = [Price ID from Stripe]
STRIPE_PRICE_ID_ENTERPRISE = [Price ID from Stripe]
```

**Storage**
```
BLOB_READ_WRITE_TOKEN = [Your Vercel Blob token]
```

### 3.3 Deploy
Click "Deploy" and wait for deployment to complete.

## Step 4: Configure Stripe Webhooks

### 4.1 Get Webhook Secret
1. Go to https://dashboard.stripe.com/webhooks
2. Click "Add endpoint"
3. Endpoint URL: `https://yourdomain.com/api/webhooks/stripe`
4. Select events:
   - `checkout.session.completed`
   - `invoice.payment_succeeded`
   - `customer.subscription.deleted`
   - `customer.subscription.updated`
5. Copy the signing secret
6. Add to Vercel as `STRIPE_WEBHOOK_SECRET`

### 4.2 Update Webhook in Vercel
1. Go to Vercel dashboard
2. Project Settings → Environment Variables
3. Update `STRIPE_WEBHOOK_SECRET` with the secret from Stripe

## Step 5: Configure Custom Domain (Optional)

### 5.1 Add Domain
1. Go to Vercel project settings
2. Click "Domains"
3. Enter your domain name
4. Follow DNS configuration instructions
5. Update `BETTER_AUTH_URL` in environment variables:
```
BETTER_AUTH_URL = https://yourdomain.com
```

## Step 8: Maintenance & Updates

### Regular Tasks
- Monitor error logs weekly
- Check database performance monthly
- Review security patches monthly
- Update dependencies quarterly
- Backup database regularly

### Scaling
If you experience high traffic:
1. Increase Neon compute resources
2. Add database replicas for read scaling
3. Consider caching layer (Redis via Upstash)
4. Monitor Stripe rate limits
5. Scale OpenAI API requests

### Cost Optimization
- Set up billing alerts
- Monitor unused resources
- Review Stripe pricing
- Optimize database queries
- Configure CDN caching headers

## Troubleshooting

### Database Connection Issues
```bash
# Test connection
psql $DATABASE_URL -c "SELECT 1"

# Check Neon dashboard for:
- Connection limits reached
- IP whitelist issues
- Query performance
```

### Stripe Integration Issues
1. Verify webhook endpoint is public
2. Check Stripe webhook logs
3. Ensure secret key is correct
4. Test webhook with Stripe CLI:
```bash
stripe listen --forward-to localhost:3000/api/webhooks/stripe
```

### OpenAI API Issues
1. Verify API key is correct
2. Check API usage limits
3. Monitor rate limits
4. Test with curl:
```bash
curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

### Storage Issues
1. Verify Blob token is correct
2. Check file permissions
3. Monitor storage quota
4. Verify CORS settings

## Rolling Back

If something goes wrong:

### Vercel Rollback
1. Go to Vercel Deployments
2. Click the previous successful deployment
3. Click "Promote to Production"

### Database Rollback
```bash
# Connect to Neon
psql $DATABASE_URL

# Check recent changes
SELECT * FROM [table_name] ORDER BY "updatedAt" DESC LIMIT 10;

# Restore from backup if needed
```
